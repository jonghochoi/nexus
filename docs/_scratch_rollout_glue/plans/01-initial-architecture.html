Phase 1 — Rollout Artifact Upload — MVP Plan

Context

학습 로깅(metric/scalar)은 이미 nexus 의 make_logger(mode="dual") 를 통해 로컬 MLflow(5100) → 중앙 MLflow 로 전송되고 있다. 다음 단계는 학습이 끝난 체크포인트를 play(추론)로 돌려 생성된 rollout 산출물을 같은 MLflow run 의 artifact 로 부착하는 것. 요구사항이 큼지막하므로(비디오, 손가락 관절 궤적, tactile, 물체 상태, 인코더 임베딩 등), MVP 는 Isaac 비디오 업로드 하나만 실제로 구현하고 나머지는 디렉토리/Python hook 시그니처만 미리 잡아두어 이후 PR 에서 채워 넣는다. observer 는 끼우지 않는다 — nexus EvalLogger 에 직접 붙는 것이 가장 minimal 하기 때문.

Architecture

train.sh ──► train.py ──► ./logs/<exp>/<run>/<algo>/{best.pth, .nexus_run.json}
                                            │
                                            │ (run_name 을 shell 이 양쪽에 동일하게 전달)
                                            ▼
play.sh  ──► play.py  ──► ./logs/<exp>/<run>/eval/<eval_id>/{videos,trajectories,...}
                              │
                              │ --upload 시
                              ▼
                          EvalLogger.from_run_info(train_dir).upload(eval_dir)
                              │
                              ▼
                          중앙 MLflow run (학습 run 에 artifact 부착)

train_and_eval.sh = train.sh && play.sh  (run_name 공유)

Directory Contract

play.py 가 --artifacts_dir 로 지정된 디렉토리에 다음 레이아웃으로 산출물을 떨군다. EvalLogger 는 이 디렉토리를 통째로 업로드한다.

./logs/<exp>/<run>/eval/<eval_id>/
├── videos/         # <span class="pill mvp">MVP</span>  rollout_*.mp4
├── trajectories/   # <span class="pill stub">stub</span> README.md (joint qpos/qvel, ee pose 포맷 명시)
├── tactile/        # <span class="pill stub">stub</span> README.md (contact pos/force 시각화 포맷)
├── objects/        # <span class="pill stub">stub</span> README.md (object pose/velocity)
├── embeddings/     # <span class="pill stub">stub</span> README.md (encoder activation .npy)
└── metrics.json    # <span class="pill mvp">MVP</span> success_rate, mean_episode_return 등

File-by-file Changes

1. play.py modify

_play_custom_algo 만 손대고 _play_rsl_rl 은 건드리지 않는다.

변경내용
새 CLI 인자 --artifacts_dir <path> · --run_info_dir <train_dir> · --upload · --video_length (기본 1000) · --num_episodes (기본 1)
video_folder 결정 --artifacts_dir 이 주어지면 artifacts_dir/videos/ 로, 아니면 기존 ./outputs/videos 로 (하위호환).
hook 시그니처 추가 play.py 안에 모듈 레벨 함수로: 
def record_videos(env, artifacts_dir): ...   # 실제 구현은 RecordVideo wrapper 가 처리하므로 no-op
def dump_trajectories(env, artifacts_dir): pass
def dump_tactile(env, artifacts_dir): pass
def dump_objects(env, artifacts_dir): pass
def dump_embeddings(agent, artifacts_dir): pass
현재는 stub. 나중에 agent.test() 루프 내부에 hook 호출을 끼워 넣는 PR 에서 채운다.
업로드 호출 agent.test() 종료 후 simulation_app.close() 이전에 EvalLogger.from_run_info(...).upload(...) 호출. (이후 phase 에서 subprocess 호출로 변경됨 — play.py 의 import graph 에 mlflow 가 끼이지 않게.)
metrics.json 생성 agent.test() 가 반환/기록하는 값이 무엇인지에 맞춰 최소 {"eval_episodes": N} 만이라도 떨궈 둠. (실제 success_rate 추출은 후속 작업)

2. train.py minimal change

본질적으로 변경 거의 없음 — make_logger(mode="dual", tb_dir=train_dir, ...) 가 이미 train_dir/.nexus_run.json 을 떨군다. 다만 sidecar 가 central URI 를 들고 가야 EvalLogger.from_run_info(target="central") 이 동작하므로 central_tracking_uri= 키워드를 한 줄 추가.

writer = make_logger(
    mode="dual",
    tb_dir=train_dir,
    run_name=run_name,
    tracking_uri="http://127.0.0.1:5100",
    central_tracking_uri="http://nexus-server:5000",   # ← 새로 추가
    ...
)

3. train.sh new

#!/usr/bin/env bash
set -euo pipefail
: "${EXPERIMENT_NAME:?required}"
: "${RUN_NAME:?required}"   # train_and_eval.sh 가 미리 계산해서 export
: "${TASK:=Sharpa-InHandRotation-Direct-v0}"
: "${ALGO:=PPO}"

python train.py \
  --task "$TASK" --algo "$ALGO" \
  --experiment_name "$EXPERIMENT_NAME" \
  --run_name "$RUN_NAME" \
  "$@"

4. play.sh new

#!/usr/bin/env bash
set -euo pipefail
: "${EXPERIMENT_NAME:?required}"
: "${RUN_NAME:?required}"
: "${TASK:=Sharpa-InHandRotation-Direct-v0}"
: "${ALGO:=PPO}"
: "${ARTIFACTS:=1}"   # 1 이면 구조화 디렉토리 생성
: "${UPLOAD:=1}"      # 1 이면 EvalLogger.upload 호출

TRAIN_DIR="./logs/${EXPERIMENT_NAME}/${RUN_NAME}/${ALGO}"
CKPT="${CKPT:-${TRAIN_DIR}/best.pth}"

EVAL_ID="$(date +%Y%m%d_%H%M%S)"
ARTIFACTS_DIR="./logs/${EXPERIMENT_NAME}/${RUN_NAME}/eval/${EVAL_ID}"

ARGS=(--task "$TASK" --algo "$ALGO" --checkpoint "$CKPT")
if [[ "$ARTIFACTS" == "1" ]]; then
  ARGS+=(--video --artifacts_dir "$ARTIFACTS_DIR")
fi
if [[ "$UPLOAD" == "1" ]]; then
  ARGS+=(--upload --run_info_dir "$TRAIN_DIR")
fi

python play.py "${ARGS[@]}" "$@"

5. train_and_eval.sh new

#!/usr/bin/env bash
set -euo pipefail
: "${EXPERIMENT_NAME:?required}"
export RUN_NAME="${RUN_NAME:-$(date +%Y-%m-%d_%H-%M-%S)}"   # ← 핵심: 양쪽이 같은 값을 봄
export EXPERIMENT_NAME

./train.sh "$@"
./play.sh                       # train.sh 의 인자(--num_envs 등)는 전달하지 않음 (의도적 분리)
왜 RUN_NAME 을 셸에서 미리 계산하는가: train.py 가 미지정 시 자체 timestamp 를 만들어버리므로, 셸이 종료 후 train_dir 을 알 방법이 없어진다. 미리 정해 양쪽에 같이 넘기면 stdout 파싱·심볼릭 링크 같은 우회를 안 해도 된다.

6. Stub README 파일들 new

play.py 가 --artifacts_dir 사용 시 자동으로 빈 카테고리 디렉토리와 README 를 떨구도록 한다 (한 함수에서 처리).

# trajectories/README.md (예시)
Joint trajectory artifacts (planned).
Format: HDF5 with datasets {qpos: (T, dof), qvel: (T, dof), ee_pose: (T, 7)}.
Filename: rollout_<ep>.h5

Critical files

  • train.py — central_tracking_uri= 한 줄 추가

Reused utilities (재사용)

  • nexus.logger.eval_logger.EvalLogger.from_run_info(path_or_dir) — sidecar 자동 매칭
  • EvalLogger.upload(eval_dir, metrics_from=..., tags=...) — 디렉토리 재귀 업로드 + index.html 자동 생성 (mp4 video 자동 임베드)
  • gym.wrappers.RecordVideo — 이미 play.py 에서 사용 중, video_folder 만 바꿔주면 됨
  • nexus.logger.make_logger(mode="dual", tb_dir=...) 가 train 시 자동으로 .nexus_run.json 작성 — train.py 변경 불필요 (sidecar 형식만)

Verification

  1. 로컬 MLflow 띄우기: mlflow server --host 127.0.0.1 --port 5100 (이미 운영 중이라면 skip).
  2. Train 단독 동작 확인: 
    EXPERIMENT_NAME=test_artifact RUN_NAME=smoke_001 ./train.sh --num_envs 64 --max_agent_steps 1000
    ./logs/test_artifact/smoke_001/PPO/.nexus_run.json 과 best.pth(또는 last.pth) 존재 확인.
  3. Play (artifact 생성만, 업로드 X): 
    EXPERIMENT_NAME=test_artifact RUN_NAME=smoke_001 UPLOAD=0 ./play.sh --num_envs 4
    ./logs/test_artifact/smoke_001/eval/<ts>/videos/*.mp4 생성 확인 + stub README 들 존재 확인.
  4. Play (업로드 포함): 
    EXPERIMENT_NAME=test_artifact RUN_NAME=smoke_001 ./play.sh --num_envs 4
    MLflow UI(5100) → 해당 run → Artifacts 탭에 videos/rollout_*.mp4 가 보이고 index.html 에서 재생되는지 확인. 중앙 MLflow 동기화 후 동일 run 에서도 보이는지 확인.
  5. End-to-end: 
    EXPERIMENT_NAME=test_artifact ./train_and_eval.sh --num_envs 64 --max_agent_steps 1000
    train + play 가 같은 RUN_NAME 으로 묶여 한 run 에 metrics + artifact 가 모두 붙는지 확인.
  6. Negative: --upload 없이 --run_info_dir 만 줘도 에러 없이 끝나야 함 (artifact 생성만).
