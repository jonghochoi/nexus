2) 3-script 분리 근거 — README 에 박는다

새 섹션 ## Why three scripts? 6줄:

  • Eval-without-train 이 일상. 같은 ckpt 를 카메라/에피소드 수/업로드 토글 바꿔가며 수십 번 다시 돌림. 합치면 재학습 강제 또는 --skip_train 플래그가 생김.
  • 실패 격리. 학습은 시간 단위, 플레이는 초~분 단위. 플레이 크래시가 학습을 무효화하면 안 됨.
  • 실행 컨텍스트가 다름. 학습은 sbatch / nohup / 장수명 tmux. 플레이는 노트북 SSH 인터랙티브. 슈퍼바이저·env 기본값이 다름.
  • 리소스 모양이 다름. 학습 --num_envs 256, 플레이 --num_envs 4 (RecordVideo 가 vec env idx=0 만 잡음). 한 entrypoint 에 자연스럽게 안 묶임.
  • CI 스모크. train.sh --max_agent_steps 100 으로 트레이너만, play.sh UPLOAD=0 으로 아티팩트 파이프라인만 검증. 합치면 깨지기 쉬운 플래그 조합 필요.
  • train_and_eval.sh 는 진짜 얇음. RUN_NAME export 후 5줄. 합치면 커지지 않고 entrypoint 가 비대해짐.

4) gym 변형 자체 개선 (observer 로 안 가도 챙기는 부분)

현재 play.py.snippet.py Block 3 의 RecordVideo 호출은 사실상 “일단 동작” 수준. 다음 7가지를 적용해 같은 gym 백엔드로도 운영가능한 수준까지 끌어올립니다.

#개선왜변경 위치
A step_trigger → episode_trigger=lambda ep: ep < args_cli.num_episodes 현재는 step==0 한 번만 트리거 → 한 개 큰 mp4. episode_trigger 로 바꾸면 에피소드 경계에서 자동 분할되어 rollout-episode-{N}.mp4 가 N 개. MLflow UI 에서 케이스별 비교 가능. Block 3
B name_prefix 에 ckpt stem + timestamp 포함 여러 번 돌릴 때 파일명 충돌·식별 불가. name_prefix=f"{Path(args_cli.checkpoint).stem}_{stamp}" 로 자기-기술적. Block 3
C videos/info.json 사이드카 mp4 만 봐서는 어떤 조건의 롤아웃인지 모름. {ckpt, seed, task, algo, num_episodes, video_length, fps, resolution, git_commit} 를 함께 떨어뜨림. upload_eval.py --metrics_from 와 별개로 인간이 읽는 메타. Block 5
D Isaac Lab 네이티브 --video --video_length --video_interval 와의 정렬 — 이미 있으면 새로 만들지 말고 위임 Isaac Lab 의 base play.py 가 동등 wrapper 를 이미 둠. 재작성하면 옵션 두 군데. 기존 인자가 있으면 그대로 forward 하고 본 스니펫의 video_length / video_fps 는 그쪽이 미지원일 때만 활성화. Block 1 (argparse 주석), Block 3
E FPS / 해상도는 RecordVideo 가 아닌 env 카메라에서 결정됨을 주석으로 명시 오해 빈발 — RecordVideo 에는 fps 인자가 없음 (env metadata 의 render_fps 사용). 해상도는 IsaacLab 의 --camera_resolution 또는 env cfg. 주석 한 줄로 함정 제거. Block 3 주석
F (opt) GIF 프리뷰 생성 — --gif_preview mp4 는 MLflow 인라인 안 됨 (그래서 EvalLogger 가 index.html 만들어줌). 부가로 2~3초 짜리 저해상 GIF 를 떨어뜨리면 Artifacts 패널에서 썸네일처럼 즉시 보임. moviepy 로 mp4→gif 한 줄. 기본 off. Block 5 (선택적)
G RecordVideo 가 vec env idx=0 만 본다는 점을 readme/주석에 명시 + num_envs=4 권장값 Train 에서 num_envs=256 으로 돌리던 사람이 그대로 play 실행하면 비디오는 1번 env 만 나옴. 일반적인 함정이라 명시 필요. play.sh 주석 + README 의 “Why three scripts?” 의 “리소스 모양이 다름” 항목과 연결

적용 후 Block 3 최종 모양:

import gymnasium as gym
import time
from pathlib import Path

# RecordVideo는 env metadata.render_fps 와 env 의 camera resolution 에서
# fps/해상도를 가져갑니다. 여기서는 변경 불가 — Isaac Lab 의
# --camera_resolution / env cfg 의 render_fps 로 조절하세요.
ckpt_stem = Path(args_cli.checkpoint).stem if args_cli.checkpoint else "rollout"
stamp     = time.strftime("%Y%m%d_%H%M%S")

env = gym.wrappers.RecordVideo(
    env,
    video_folder=str(video_folder),
    episode_trigger=lambda ep: ep < args_cli.num_episodes,   # ← A
    video_length=args_cli.video_length,
    name_prefix=f"{ckpt_stem}_{stamp}",                       # ← B
    disable_logger=True,
)
